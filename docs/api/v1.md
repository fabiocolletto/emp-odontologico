# API v1 — Contratos Iniciais

## Convenções Gerais
- Base path: `/v1`
- Auth: `Authorization: Bearer <token>`
- Content-Type: `application/json`
- Multi-tenant: `clinicId` no token e validado no backend.

## Mapeamento de Estado Local para Recursos
- `allPatients` → `GET/POST /v1/patients`
- `allProcedures` → `GET/POST /v1/procedures`
- `APPOINTMENTS` → `GET/POST /v1/appointments`

---

## Patients

### GET /v1/patients
Retorna lista de pacientes da clínica autenticada.

**Response 200**
```json
[
  {
    "id": "pat_001",
    "name": "Ana Paula Souza",
    "phone": "(11) 98877-6655",
    "birthDate": "1992-05-15",
    "gender": "Feminino",
    "lastVisitAt": "2026-04-12T15:00:00Z"
  }
]
```

### POST /v1/patients
Cria paciente.

**Request**
```json
{
  "name": "Ricardo Albuquerque",
  "phone": "(11) 97766-5544",
  "birthDate": "1985-10-22",
  "gender": "Masculino"
}
```

---

## Procedures

### GET /v1/procedures
Retorna catálogo de procedimentos ativos.

### POST /v1/procedures
Cria novo procedimento.

**Request**
```json
{
  "name": "Clareamento Dental"
}
```

---

## Appointments

### GET /v1/appointments
Retorna agenda da clínica (filtro por data opcional).

### POST /v1/appointments
Cria consulta.

**Request**
```json
{
  "patientId": "pat_001",
  "procedureId": "proc_001",
  "startsAt": "2026-04-18T14:15:00Z",
  "notes": "Primeira avaliação"
}
```

---

## Erros
Formato padrão:
```json
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Campo obrigatório ausente",
    "details": ["name is required"]
  }
}
```

## Códigos comuns
- `400` validação
- `401` não autenticado
- `403` sem permissão
- `404` não encontrado
- `409` conflito
- `500` erro interno
