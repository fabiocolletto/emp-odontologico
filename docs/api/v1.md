# API v1 — Contratos Iniciais

## Convenções Gerais
- Base path: `/v1`
- Auth: `Authorization: Bearer <token>`
- Content-Type: `application/json`
- Multi-tenant: o backend usa `public.current_clinic_id()` para escopo ativo por usuário.

## Mapeamento de Estado Local para Recursos
- `allPatients` → `GET/POST /v1/patients`
- `allProcedures` → `GET/POST /v1/procedures`
- `APPOINTMENTS` → `GET/POST /v1/appointments`

## Clinics

### GET /v1/clinics
Retorna as clínicas vinculadas ao usuário autenticado.

**Response 200**
```json
[
  {
    "id": "11111111-1111-1111-1111-111111111111",
    "name": "Clínica Sorriso Centro",
    "legalName": "Sorriso Centro Odontologia LTDA",
    "taxId": "12345678000195",
    "timezone": "America/Sao_Paulo",
    "phone": "(11) 3333-4444",
    "email": "contato@sorrisocentro.com",
    "ownerUserId": "aaaaaaaa-aaaa-aaaa-aaaa-aaaaaaaaaaa1",
    "isActive": true
  }
]
```

### POST /v1/clinics
Cria uma nova clínica com owner obrigatório (`ownerUserId = auth.uid()`).

**Request**
```json
{
  "name": "Nova Clínica Odonto",
  "legalName": "Nova Clínica Odonto LTDA",
  "taxId": "98765432000110",
  "timezone": "America/Sao_Paulo",
  "phone": "(11) 3555-9000",
  "email": "contato@novaodonto.com",
  "ownerUserId": "aaaaaaaa-aaaa-aaaa-aaaa-aaaaaaaaaaa1"
}
```

**Response 201**
```json
{
  "id": "44444444-4444-4444-4444-444444444444",
  "name": "Nova Clínica Odonto",
  "ownerUserId": "aaaaaaaa-aaaa-aaaa-aaaa-aaaaaaaaaaa1"
}
```

### PATCH /v1/clinics/:id
Atualiza dados de contato e fiscais da clínica ativa (ex.: `legalName`, `taxId`, `phone`, `email`, `timezone`).

**Request**
```json
{
  "legalName": "Nova Clínica Odonto e Estética LTDA",
  "taxId": "98765432000110",
  "phone": "(11) 3555-9001",
  "email": "financeiro@novaodonto.com"
}
```

### POST /v1/clinics/switch
Troca a clínica ativa no contexto do usuário autenticado.

**Request**
```json
{
  "clinicId": "22222222-2222-2222-2222-222222222222"
}
```

**Response 204**
Sem corpo.

## Regras de autorização

- `owner` e `admin` podem editar dados sensíveis da clínica (contato/fiscais), desde que seja a clínica ativa do contexto.
- O `owner` da clínica precisa ser o usuário autenticado na criação (`ownerUserId = auth.uid()`).
- O usuário sempre precisa estar vinculado a pelo menos uma clínica (com `staff_profile` ativo e `user_clinic_context` válido).
- Troca de clínica ativa (`POST /v1/clinics/switch`) só é permitida se o usuário fizer parte da clínica destino.

## Erros
Formato padrão:
```json
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Campo obrigatório ausente",
    "details": ["name is required"]
  }
}
```

## Códigos comuns
- `400` validação
- `401` não autenticado
- `403` sem permissão
- `404` não encontrado
- `409` conflito
- `500` erro interno
